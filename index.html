Git command list
Command Explanation
git config --global user.name Задати ім’я
git config --global user.email Задати e-mail
git init Запуск команди в каталозі на вашому комп'ютері створить там новий .git підкаталог. Це власне локальне сховище Git і воно містить всю структуру та метадані, що складають сховище Git
git status Допомагає зрозуміти поточний стан вашої локальної робочої копії. Він відобразить будь-які зміни у ваших локальних файлах, які ви ще не зафіксували у сховищі. Крім того, він також повідомить вам, яка гілка на даний момент активна, і якщо ця гілка має якісь несинхронізовані коміти порівняно із віддаленим аналогом.
git add Позначає зміни, які будуть включені до наступного коміту. Це додає зміни до "Staging Area" Git, вміст якої потім можна обернути в новій редакції за допомогою команди "git commit".
git commit Використовується для збереження змін у локальному сховищі. Зауважте, що перед запуском команди "git commit" вам потрібно чітко вказати Git, які зміни ви хочете включити у коміт. Це означає, що файл не буде автоматично включений до наступного коміту лише тому, що його було змінено. Натомість вам потрібно використовувати команду "git add", щоб позначити бажані зміни для включення. Також зауважте, що в Git, коміт не передається автоматично на віддалений сервер. Використання команди "git commit" зберігає лише новий об'єкт коміту у локальному сховищі Git. Обмін комітами повинен виконуватися вручну та явно (за допомогою команд "git fetch", "git pull" та "git push").
git log Показує журнали комітів.
git checkout Може переключити поточну активну гілку, але вона також може бути використана для відновлення файлів. Найбільш поширеним варіантом використання "checkout" є перехід на іншу гілку , що робить її новою гілкою HEAD. Інший варіант використання "checkout" - це коли ви хочете відновити історичну версію певного файлу. Таким чином, ви можете скинути окремі файли до попередніх версій, залишаючи решту проекту без змін.
git branch Допомагає створювати, видаляти та перераховувати гілки. Це команда go-to, коли справа доходить до управління будь-яким аспектом ваших гілок - незалежно від того, в локальному сховищі чи віддалено.
git push Використовується для публікації нових локальних комітів на віддаленому сервері. Зверніть увагу, що команду "push" також можна використовувати для видалення віддаленої гілки.
git pull Використовується для завантаження та інтеграції віддалених змін.
git merge Злиття певних гілок проекту
git clone Завантажує існуюче сховище Git на ваш локальний комп'ютер. Потім ви отримаєте повномасштабну локальну версію цього репозиторію Git і зможете почати працювати над проектом. Як правило, "оригінальне" сховище розташоване на віддаленому сервері, часто від такої служби, як GitHub, Bitbucket або GitLab). Потім URL-адресу цього віддаленого сховища називають "джерелом".
git revert Скасувати деякі існуючі коміти
git reset Скинути поточний HEAD до заданого стану
git rebase Перебазувує поточний HEAD у branch Не перебазуйте опубліковані коміти Перебазування - це процес переміщення послідовності коммітов до нової базової коммітов або їх об'єднання.
git cherry-pick Дозволяє інтегрувати окремі коміти у поточну гілку HEAD - замість того, щоб інтегрувати всі коміти філії (наприклад, з merge та rebase ).
git stash Може допомогти вам (тимчасово, але безпечно) зберігати нездійснені локальні зміни - і залишити вам чисту робочу копію.

.gitignore - визначає навмисно не відстежувані файли, які потрібно ігнорувати. Детальніше тут
